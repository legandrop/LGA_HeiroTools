BUG REPORT: Foundry Nuke Studio 16.08 - openInTimeline() Instability

SUMMARY:
hiero.ui.openInTimeline() causes Nuke Studio to crash immediately when called in Nuke Studio 16.08.

REPRODUCTION STEPS:
1. Open any project with sequences in Nuke Studio 16.08
2. Run the minimal script
3. Result: Nuke Studio CRASHES immediately (silent C++ crash)

EXPECTED BEHAVIOR (Nuke Studio 15.x):
- Timeline/viewer created successfully
- Nuke Studio remains stable

ACTUAL BEHAVIOR (Nuke Studio 16.08):
- Nuke Studio crashes immediately when openInTimeline() is called
- Crash is 100% reproducible

VERSIONS TESTED:
- Nuke Studio 15.x (Windows/macOS): ✅ Works perfectly
- Nuke Studio 16.08 (Windows/macOS): ❌ Bug reproducible 100%

VERSION SPECIFICITY:
- Affects version 16.08 on both Windows and macOS platforms
- Previous version (15.x) works fine on both platforms

TECHNICAL DETAILS:
- Problem occurs with:
  • hiero.ui.openInTimeline()
  • hiero.ui.openInViewer()
- Problem does NOT occur with:
  • hiero.ui.getTimelineEditor() (when timeline not shown)
- Problem is NOT related to processEvents()
- Affects ALL programmatic timeline/viewer creation methods
- Results in immediate silent C++ crash when calling the function

EVIDENCE:
- Tested with isolated function calls
- Bug is 100% reproducible in version 16.08
- Minimal test case provided in accompanying Python script
